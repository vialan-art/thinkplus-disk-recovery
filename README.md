# thinkplus 移动硬盘修复记录（数据重要版）

## 设备信息

| 项目 | 内容 |
|------|------|
| 硬盘 | thinkplus 移动硬盘 |
| 容量 | 1TB |
| 当前文件系统 | exFAT |
| 分区名 | thinkplus |
| Mac 设备路径 | `/dev/disk5s1` |
| Mac 挂载路径 | `/Volumes/thinkplus` |

## 问题现象

1. 插入硬盘后 Finder 无响应、崩溃
2. 拔掉硬盘后 Finder 立即恢复正常
3. Mac 曾因硬盘卡死而强制重启
4. 命令行读取部分目录时挂起：`du`、`find`、`diskutil unmountDisk` 均超时
5. 尝试用 rclone 上传到 Google Drive 时速度为 0，无法读取

## 已尝试的修复（Mac 端）

| 操作 | 结果 |
|------|------|
| 重启 Finder | 无效 |
| 强制卸载硬盘 | 超时失败 |
| rclone 备份到 Google Drive | 速度为 0，失败 |
| 等待 macOS 后台自动修复 | 几小时后仍无改善 |

## 当前判断

硬盘文件系统已损坏，macOS 对损坏 exFAT 的修复能力有限。由于数据非常重要，**先抢救数据，再修复硬盘**。

## 方案一：Windows 电脑急救（推荐，优先执行）

### 第一步：安全复制最重要文件

在运行任何修复命令前，先手动复制最关键的文件到 Windows 本地或另一个移动硬盘/U盘：

1. 插入硬盘
2. 打开 `此电脑`，查看硬盘盘符（例如 `E:`）
3. 打开硬盘，进入以下目录，优先复制能正常打开的文件夹：
   - `以前`
   - `资源`
   - `iphone`
   - `iCloud照片.7z`
   - `2025双旦北海道东京`
   - `日本行`
   - `堆栈练习`
4. 复制到 Windows 桌面或另一个外置盘

> **原则**：先不运行任何修复，能复制多少算多少。

### 第二步：运行 chkdsk 修复

1. 按 `Win + R`，输入 `cmd`，按 `Ctrl + Shift + Enter` 以管理员身份运行
2. 在命令提示符中输入（将 `E:` 替换为实际盘符）：

   ```cmd
   chkdsk E: /f
   ```

3. 如果提示「卷正在使用，是否计划在下次重启时检查」，输入 `Y` 回车
4. 等待扫描并修复完成。大容量硬盘可能需要 30 分钟到数小时
5. 修复完成后，重新打开硬盘检查文件是否完整

### 第三步：chkdsk 修复后的文件检查

- 检查是否出现 `FOUND.000`、`FOUND.001` 等文件夹
- 这些文件夹里的 `FILE0000.CHK`、`FILE0001.CHK` 是被恢复但丢失文件名的文件
- 尝试用对应软件打开它们（图片改后缀为 `.jpg`/`.png`，视频改 `.mp4`/`.mov`）

### 第四步：备份到云端

1. 登录 Google Drive 网页版或安装 Google Drive 客户端
2. 将修复后的文件上传到 Google Drive（5TB 空间足够）
3. 或上传到夸克网盘（3.3TB 空间足够）

### 第五步：格式化硬盘（修复成功后）

1. 右键硬盘 → 格式化
2. 文件系统选择 **exFAT**（兼容 Mac 和 Windows）
3. 分配单元大小选默认
4. 勾选「快速格式化」，开始

## 方案二：如果 chkdsk 失败或文件仍无法读取

### 使用专业数据恢复软件（Windows 端）

按推荐顺序尝试：

| 软件 | 特点 | 备注 |
|------|------|------|
| R-Studio | 专业级，支持 exFAT/NTFS，恢复率高 | 付费，有试用版 |
| DMDE | 功能强大，有免费版 | 免费版有恢复容量限制 |
| Recuva | 免费，操作简单 | 对严重损坏效果有限 |
| DiskGenius | 国产，有免费版 | 中文界面 |

操作步骤：
1. 下载并安装软件
2. 选择 thinkplus 硬盘
3. 运行「完整扫描」或「深度扫描」
4. 扫描完成后，预览能识别的文件
5. 将文件恢复到另一个硬盘（不要恢复到原硬盘）
6. 确认恢复完成后，再格式化原硬盘

## 方案三：Mac 端最后尝试（如果只有 Mac 可用）

如果临时找不到 Windows 电脑，可以按以下步骤在 Mac 上尝试：

### 1. 重新插上硬盘

### 2. 立刻强制卸载

```bash
sudo umount -f /Volumes/thinkplus
```

### 3. 运行 fsck_exfat 检查

```bash
sudo fsck_exfat -d /dev/disk5s1
```

### 4. 如果报告错误，执行修复

```bash
sudo fsck_exfat -y /dev/disk5s1
```

### 5. 修复完成后重新挂载

```bash
sudo mkdir -p /Volumes/thinkplus-fixed
sudo mount -t exfat /dev/disk5s1 /Volumes/thinkplus-fixed
```

### 6. 尝试复制文件

```bash
rsync -av --progress /Volumes/thinkplus-fixed/以前 ~/Desktop/
```

> **警告**：Mac 端 fsck_exfat 修复能力较弱，重要数据优先在 Windows 上处理。

## 绝对禁止事项

- ❌ 不要反复插拔硬盘
- ❌ 不要在修复前格式化硬盘
- ❌ 不要把恢复出来的文件保存回原硬盘
- ❌ 不要让硬盘在修复过程中断电或断开
- ❌ 不要运行来历不明的「硬盘修复工具」

## 修复成功后的维护建议

1. 格式化硬盘为 exFAT
2. 定期将重要数据备份到云端
3. 避免在传输数据时移动硬盘或触碰数据线
4. 每次拔出硬盘前使用「安全删除硬件」
