# thinkplus 移动硬盘数据抢救手册

## 设备信息

| 项目 | 内容 |
|------|------|
| 硬盘 | thinkplus 移动硬盘 |
| 容量 | 1TB |
| 当前文件系统 | exFAT |
| 分区名 | thinkplus |
| Mac 设备路径 | `/dev/disk5s1` |
| Mac 挂载路径 | `/Volumes/thinkplus` |

## 问题现象

1. 插入硬盘后 Finder 无响应、崩溃
2. 拔掉硬盘后 Finder 立即恢复正常
3. Mac 曾因硬盘卡死而强制重启
4. 命令行读取部分目录时挂起：`du`、`find`、`diskutil unmountDisk` 均超时
5. 尝试用 rclone 上传到 Google Drive 时速度为 0，无法读取

## 已尝试的修复（Mac 端）

| 操作 | 结果 |
|------|------|
| 重启 Finder | 无效 |
| 强制卸载硬盘 | 超时失败 |
| rclone 备份到 Google Drive | 速度为 0，失败 |
| 等待 macOS 后台自动修复 | 几小时后仍无改善 |

## 核心原则（必读）

1. **先判断是否存在硬件故障。**
2. **对于重要数据，优先制作磁盘镜像，而不是直接修复。**
3. **所有恢复操作尽量针对镜像进行。**
4. **将 `chkdsk` 作为恢复后的修复手段，而不是优先步骤。**
5. **增加停止条件和恢复验证步骤，避免因持续尝试造成二次损坏。**

## 第一步：判断是否为硬件故障

在继续之前，先观察以下症状：

| 检查项 | 正常 | 异常（可能是硬件故障） |
|------|------|------------------|
| 硬盘声音 | 轻微转动声 | 咔哒声、点击声、反复启停 |
| 容量识别 | 显示 1TB | 容量显示异常、不识别 |
| 连接稳定性 | 稳定连接 | 频繁自动断开重连 |
| 读取速度 | 能正常读取 | 持续为 0 MB/s、越来越慢 |
| SMART | 无警告 | Reallocated Sector / Pending Sector 异常 |

### Windows 检查 SMART

下载 [CrystalDiskInfo](https://crystalmark.info/en/software/crystaldiskinfo/)，查看硬盘健康状态。

### Mac/Linux 检查 SMART

```bash
smartctl -a /dev/disk5
```

> 注意：很多 USB 转 SATA 桥接芯片不支持 SMART。

### 如果怀疑硬件故障

立即停止所有修复操作，优先制作磁盘镜像。不要反复插拔、扫描、重启。

---

## 第二步：抢救能直接读取的数据（最高优先级）

在运行任何修复或扫描之前，先手动复制还能打开的文件。

### Windows 操作

1. 插入硬盘
2. 打开 `此电脑`，查看硬盘盘符（例如 `E:`）
3. 进入以下目录，优先复制能正常打开的文件夹：
   - `以前`
   - `资源`
   - `iphone`
   - `iCloud照片.7z`
   - `2025双旦北海道东京`
   - `日本行`
   - `堆栈练习`
4. 复制到 Windows 桌面、另一个外置硬盘或 U 盘

### Mac 操作（如果 Finder 卡死）

不要使用 Finder。使用命令行：

```bash
rsync -av --progress "/Volumes/thinkplus/以前" ~/Desktop/
```

或使用 `cp`、`ditto`、`tar`。

> **原则**：能复制多少算多少，不要先修复。

---

## 第三步：制作整盘镜像（数据价值极高时）

如果硬盘还能被系统识别，建议先制作镜像。之后所有恢复都针对镜像进行。

### 推荐工具

| 工具 | 平台 | 特点 |
|------|------|------|
| GNU ddrescue | Linux/macOS | 免费，支持断点续传，可跳过坏块 |
| HDDSuperClone | Windows/Linux | 专业级，适合坏道盘 |
| R-Studio | Windows/Mac | 商业软件，可直接创建磁盘镜像 |

### Linux/macOS 使用 ddrescue 示例

```bash
sudo apt install gddrescue    # Linux
brew install ddrescue          # macOS

sudo ddrescue -d -r3 /dev/disk5 ~/Desktop/thinkplus.img ~/Desktop/thinkplus.log
```

参数说明：
- `-d`：直接读取设备
- `-r3`：遇到错误重试 3 次
- 第一个路径：源设备
- 第二个路径：镜像保存位置（需要足够大的空间）
- 第三个路径：日志文件，支持断点续传

> **注意**：镜像文件需要存放在另一个容量大于 1TB 的硬盘上，不要保存到原盘或 Mac 本地。

---

## 第四步：使用专业恢复软件扫描

如果直接复制失败，或需要恢复已删除/损坏的文件，使用专业恢复软件。

### 推荐排序

| 推荐 | 软件 | 特点 |
|------|------|------|
| ⭐⭐⭐⭐⭐ | R-Studio | 恢复率最高，商业软件 |
| ⭐⭐⭐⭐⭐ | DMDE | 专业级，价格低 |
| ⭐⭐⭐⭐ | DiskGenius | 国内支持好，中文界面 |
| ⭐⭐⭐ | UFS Explorer | 非常强，但价格高 |
| ⭐⭐ | Recuva | 适合误删，不适合严重损坏 |

### 操作步骤

1. 安装恢复软件
2. 选择 thinkplus 硬盘或之前制作的镜像文件
3. 运行「完整扫描」或「深度扫描」
4. 扫描完成后，预览能识别的文件
5. **将文件恢复到另一个硬盘**（不要恢复到原硬盘）
6. 确认恢复完成后，再考虑格式化原硬盘

---

## 第五步：chkdsk 修复原盘（最后一步）

当数据已经抢救完成或确认无法恢复后，才运行 `chkdsk` 修复原盘。

> **警告**：`chkdsk /f` 会修复文件系统一致性，过程中可能删除损坏的目录、索引或目录项。它可能让还能恢复的文件彻底丢失。

### Windows 操作

1. 按 `Win + R`，输入 `cmd`
2. 按 `Ctrl + Shift + Enter` 以管理员身份运行
3. 输入（将 `E:` 替换为实际盘符）：

   ```cmd
   chkdsk E: /f
   ```

4. 等待扫描并修复完成
5. 修复完成后检查是否出现 `FOUND.000`、`FOUND.001` 等文件夹
6. 这些文件夹里的 `FILE0000.CHK`、`FILE0001.CHK` 是丢失文件名的恢复文件，可尝试改后缀后打开

---

## 第六步：备份到云端

修复并确认文件完整后，立即备份：

- **Google Drive**：5TB 空间，已配置
- **夸克网盘**：3.3TB 剩余空间

---

## 第七步：格式化硬盘

确认数据已全部备份后，格式化硬盘：

1. 右键硬盘 → 格式化
2. 文件系统选择 **exFAT**（兼容 Mac 和 Windows）
3. 分配单元大小选默认
4. 勾选「快速格式化」，开始

---

## 什么时候必须停止

出现以下任一情况，立即停止所有操作：

- 硬盘发出咔哒声、点击声等异响
- 容量识别异常
- 频繁自动断开重连
- 读取速度越来越慢
- 每次扫描都卡住
- SMART 显示重映射扇区或待处理扇区异常

这些可能是硬件故障征兆，继续操作可能造成二次损坏。

---

## 恢复后的验证

恢复完成以后，不要只看文件是否存在，还应该验证：

- 图片是否能正常打开
- 视频能否完整播放到结尾
- 压缩包能否正常解压
- 文档是否能正常打开
- 如果有历史备份，对比 SHA256 校验值

很多恢复软件能恢复文件名，但文件内容可能已经损坏。
